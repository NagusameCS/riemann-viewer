Riemannian Manifold Viewer — Guide

What am I looking at?

This viewer visualizes the weight space geometry of neural networks as a Riemannian manifold. Each model's learned parameters live in a high-dimensional space. We project them to 3D via PCA and compute differential-geometric quantities to understand the shape of the "landscape" the model has learned.

Definitions

  • Riemannian Manifold — A smooth surface equipped with a way to measure distances, angles, and curvature at every point. Here, the "surface" is the space of model weights.
  • Scalar Curvature (R) — A single number at each point that summarizes how curved the manifold is there. R < 0 = saddle-like (hyperbolic), R > 0 = sphere-like, R = 0 = flat.
  • Metric Tensor (g) — Defines how to measure infinitesimal distances on the manifold. The diagonal entries gii control stretching along each PCA axis. Visualized as ellipsoids.
  • Christoffel Symbols (Γ) — Describe how coordinate axes "twist" as you move across the manifold. ‖Γ‖ is their norm — higher means more connection curvature. Controls the size of curvature field points.
  • PCA Point Cloud — The raw 3D projection (via Principal Component Analysis) of weight vectors from different layers of the model. No curvature data — just spatial positions.
  • Curvature Field — Sampled points where R, g, and Γ were actually computed. These are the data points the surface is built from.
  • Geodesic — The shortest path between two points on the manifold (like a great circle on a sphere).
  • Intrinsic Dimension — How many independent directions the weight space actually uses (often much less than the full parameter count).

Color Scheme

  • The curvature surface is colored by scalar curvature R: blue tones = negative (hyperbolic), warm tones = positive (elliptic), neutral = near zero.
  • All other elements (point cloud, connections, wireframe, ellipsoids) are rendered in grayscale to keep focus on the curvature.
  • When all curvature values are identical (R≈0), coloring falls back to Christoffel norm — from cool to warm based on ‖Γ‖.

How Inference Works (Step by Step)

When a prompt like "The capital of France is" passes through the model:

  1. Tokenization — The text is split into tokens (subword pieces). Each token maps to an embedding vector — a point in the high-dimensional weight space.
  2. Layer-by-layer transform — The embedding travels through each transformer layer. At each layer, the attention mechanism and feed-forward network transform it. Geometrically, the point "slides" along the manifold — following a path determined by the weights at that layer.
  3. Attention as parallel transport — Self-attention can be seen as parallel-transporting the token's representation along the manifold. The Christoffel symbols Γ govern how this transport works — they encode the "curvature" of the path.
  4. Curvature = information density — Regions where |R| is high correspond to layers/weight regions where the model makes sharp representational changes. Flat regions (R≈0) suggest smooth, gradual transformation.
  5. Output projection — The final representation is projected to vocabulary space, producing logits for the next token. The model predicts: "Paris".

The manifold you see here captures the static geometry of the weight space — the "terrain" over which all such inference paths travel.

Controls

  • Orbit — Left-click + drag to rotate
  • Zoom — Scroll wheel (can zoom very close)
  • Pan — Right-click + drag
  • Settings ⚙ — Toggle layers, adjust subdivision, displacement, opacity
  • Subdivision — Controls mesh resolution (0 = raw triangles, 4 = very smooth). Rebuilds the surface when changed.
  • Compare — Load multiple models via Settings → Add Model, or drag-and-drop JSON files
